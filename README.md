# Sulpak Review Moderator

Автоматическая модерация отзывов на товары интернет-магазина [Sulpak](https://sulpak.kz) с помощью Claude AI.

Поддерживает:
- **Модерацию текста** — Claude решает, публиковать ли отзыв (approve / reject / manual_review)
- **Vision-проверку фото** — Claude смотрит на приложённые к отзыву картинки и оценивает, соответствуют ли они товару

## Как работает

```
Email (Exchange) → Парсинг HTML → Claude text API → Публикация текста
                                          ↓
                              Claude vision API → Аннотация медиа (без публикации)
```

1. Скрипт подключается к Exchange (EWS) и читает непрочитанные письма с отзывами
2. Парсит HTML-письмо: имя, оценку, плюсы/минусы, текст, ссылки, **прямые URL на медиа-файлы**
3. Отправляет текст отзыва в Claude API для решения `approve / reject / manual_review`
4. Если текст одобрен и `AUTO_PUBLISH=true` — **публикует текст** отзыва
5. Если к отзыву приложены фото и текст опубликован — гоняет фото через **Claude vision** и записывает вердикт в категорию письма и JSONL-лог (но не публикует фото)
6. Помечает письмо категорией в Exchange для модератора

## Режимы работы

| Режим | `AUTO_PUBLISH` | Поведение |
|-------|---------------|-----------|
| **Помощник** | `false` | Только помечает письма категориями, ничего не публикует |
| **Автомат** | `true` | Публикует текст одобренных отзывов с `confidence ≥ CONFIDENCE_THRESHOLD` |

## Логика автопубликации

| Случай | Публикация текста | Vision по фото | Категория письма |
|--------|------------------|----------------|------------------|
| `approve`, `conf ≥ 0.85`, **без медиа** | ✅ да (при `AUTO_PUBLISH=true`) | не вызывается | `✅ Опубликовано ИИ` |
| `approve`, `conf ≥ 0.85`, **с медиа** | ✅ да (при `AUTO_PUBLISH=true`) | ✅ вызывается | `✅ Опубликовано ИИ \| 📷 фото ок / ⚠️ / ❌ N%` |
| `approve`, `conf < 0.85`, **с медиа** | нет | ✅ вызывается | `✅ Одобрено ИИ \| 📷 фото ...` |
| `approve`, `conf < 0.85`, **без медиа** | нет | — | `✅ Одобрено ИИ` |
| `reject` | нет | нет | `❌ Отклонено ИИ` |
| `manual_review` | нет | нет | `⚠️ Проверить вручную` |

Vision по медиа вызывается **для всех `approve` с фото, независимо от режима и порога confidence** — чтобы модератор всегда видел качество прикреплённых фото в ежедневном отчёте и категории письма.

Медиа сами на сайт **не публикуются** — vision только помечает категорию письма и пишет вердикт в JSONL, чтобы модератор видел, нужно ли вмешаться (например, удалить неуместное фото из CMS).

Если vision-вердикт = `reject` (на фото мусор / персональные данные / чужой товар), письмо остаётся непрочитанным с high importance — сигнал модератору.

## Решения текстовой модерации

| Решение | Описание |
|---------|----------|
| `approve` | Отзыв о товаре, информативный, без нарушений |
| `reject` | Спам, нецензурщина, отзыв о сервисе, персональные данные |
| `manual_review` | Пограничный случай, жалоба на брак, неясный текст |

## Решения vision-модерации (по фото)

| Vision-вердикт | Когда |
|----------------|-------|
| `approve` | На фото действительно приложенный товар, контент цивильный |
| `reject` | Нет товара (котик, пейзаж, еда), скриншот чата/чека, реклама, непристойный контент |
| `manual_review` | Товар может быть, но уверенности нет; виден дефект — нужна человеческая оценка |

## Правила модерации

### Отклонить:
- Отзыв о сервисе/доставке, а не о товаре (>70% текста о сервисе)
- Спам, реклама, ссылки на сторонние сайты
- Нецензурная лексика, оскорбления
- Персональные данные (телефоны, адреса, ФИО сотрудников)
- Слишком короткий/неинформативный отзыв

### Одобрить:
- Описание опыта использования товара
- Конкретные характеристики, плюсы/минусы
- Допускается краткое упоминание сервиса (1-2 предложения)

### На ручную проверку:
- Пограничный контент (50/50 товар/сервис)
- Жалоба на брак/дефект
- Текст на другом языке или неразборчивый

```bash

git clone https://github.com/<username>/sulpak-reviews.git
```

## Vision-проверка медиа

К письму с отзывом могут быть приложены ссылки вида `Медиа 1`, `Медиа 2`, ведущие на CDN-хранилище Sulpak (`object.pscloud.io`). Скрипт автоматически:

1. Собирает прямые URL в `review.media_urls`
2. Фильтрует поддерживаемые форматы: `.jpg`, `.jpeg`, `.png`, `.webp`, `.gif` (видео, YouTube Shorts и сторонние ссылки складываются в `skipped_urls`)
3. Отправляет в Claude Sonnet 4 (vision) вместе с текстом отзыва и URL товара — картинки передаются прямо по URL, без скачивания
4. Получает JSON-вердикт с пер-картиночной разбивкой (`per_image: [{index, verdict, note}]`)

Vision запускается **для каждого отзыва, где текст получил `approve` и есть медиа**, независимо от `AUTO_PUBLISH` и порога confidence — чтобы модератор всегда видел оценку фото в отчёте.

### Возможные вердикты vision

| Вердикт | Значение | Действие |
|---------|----------|----------|
| `approve` | Фото соответствуют товару, контент цивильный | Текст публикуется (если прошёл текстовую проверку), категория письма: `... \| 📷 фото ок N%` |
| `reject` | Мусор, не товар, персональные данные, реклама | Текст публикуется, но письмо остаётся непрочитанным с high importance: `... \| 📷 фото ❌ N%` → модератор удаляет фото в CMS |
| `manual_review` | Товар возможно есть, но уверенности нет; виден дефект | Категория `... \| 📷 фото ⚠️ N%` |

### Пример JSONL-записи отзыва с медиа

См. [`docs/sample_log.jsonl`](docs/sample_log.jsonl) — 4 реалистичных записи: без медиа, с медиа и vision=ok, с медиа и vision=reject (кот + скан чека), текст=reject.

Сокращённая выжимка записи с медиа:

```json
{
  "name": "Айдар",
  "decision": "approve",
  "confidence": 0.91,
  "has_media": true,
  "media_count": 3,
  "media_urls": [
    "https://object.pscloud.io/cms/cms/CustomersUploads/com60162_cl83_ob2205_1.jpeg",
    "..."
  ],
  "vision": {
    "decision": "approve",
    "confidence": 0.95,
    "reason": "Фото демонстрируют наушники Acron A3 ...",
    "per_image": [
      {"index": 1, "verdict": "ok", "note": "зарядный кейс наушников Acron в руке"},
      {"index": 2, "verdict": "ok", "note": "открытая коробка с кейсом и кабелем"},
      {"index": 3, "verdict": "ok", "note": "упаковка Acron T10 TRUE Wireless"}
    ]
  },
  "usage": {
    "text":   {"input_tokens": 498, "output_tokens": 102},
    "vision": {"input_tokens": 4120, "output_tokens": 245}
  },
  "published": true
}
```

Пер-картиночные `verdict` используются vision для обоснования решения:
- `ok` — фото товара в норме
- `not_product` — на фото не товар (животное, пейзаж, еда…)
- `personal_data` — скриншот чата, чек, паспорт
- `other` — неприемлемый контент (реклама, непристойное и т. п.)

### Пример ежедневного отчёта

См. [`docs/sample_report.html`](docs/sample_report.html) — полностью сгенерированный отчёт на основе `sample_log.jsonl`. Открыть в браузере.

В отчёт входят:
- Таблица решений (approve / reject / manual_review) с процентами
- Разбивка публикации: без медиа / с медиа
- Блок **📷 Медиа в отзывах** — счётчики vision-вердиктов
- Таблица **❌ Медиа требуют ручной чистки в CMS** — со ссылками на товары
- Блок **💰 Стоимость Claude API** — токены и USD отдельно по текстовой модерации и vision
- Причины отклонения, ручная проверка, отклонённые отзывы

### Ручной прогон vision

```bash
py test_media_vision.py 3   # проверить 3 последних отзыва с медиа
```

Ничего не публикует и не меняет состояние писем — только печатает вердикт в консоль. Удобно для отладки и анализа пограничных случаев.

## Ежедневный отчёт

Скрипт `daily_report.py` собирает статистику из JSONL-логов и отправляет HTML-отчёт на email:

- Таблица: одобрено / отклонено / на проверку (с процентами)
- Опубликовано: разбивка по наличию медиа (без медиа / с медиа)
- Блок «📷 Медиа в отзывах»: vision-вердикты (ок / на проверку / отклонено)
- Таблица «Медиа требуют ручной чистки в CMS» — для случаев vision-reject
- Блок «💰 Стоимость Claude API» — токены и траты за день, отдельно текст и vision
- Причины отклонения (топ)
- Список отзывов на ручную проверку
- Список отклонённых с причинами

```bash
# Отчёт за сегодня
py daily_report.py

# Отчёт за конкретную дату
py daily_report.py 20260313
```

## Расписание (Windows Task Scheduler)

| Задание | Расписание | Описание |
|---------|-----------|----------|
| `SulpakReviewModeration` | Каждые 2 часа с 9:00 | Обработка новых отзывов |
| `SulpakDailyReport` | Ежедневно в 19:00 | Сводка за день на email |

### Управление

```cmd
:: Запустить модерацию вручную
schtasks /run /tn "SulpakReviewModeration"

:: Запустить отчёт вручную
schtasks /run /tn "SulpakDailyReport"

:: Посмотреть статус
schtasks /query /tn "SulpakReviewModeration"
schtasks /query /tn "SulpakDailyReport"
```

## Логи

- `moderation_YYYYMMDD.log` — полный лог работы (текстовый)
- `moderation_log_YYYYMMDD.jsonl` — результаты в JSON Lines (для отчётов)

## Модель AI

Используется **Claude Sonnet 4** (`claude-sonnet-4-20250514`) через Anthropic API — и для текстовой модерации, и для vision.

Цены (на апрель 2026):
- Input: $3 / 1M токенов
- Output: $15 / 1M токенов
- Cache read: $0.30 / 1M, cache write: $3.75 / 1M

Типичная стоимость:
- Текстовая модерация отзыва — ~$0.001–0.002
- Vision-проверка отзыва с 3 фото — ~$0.01–0.02

Фактические траты за день видны в блоке «💰 Стоимость Claude API» ежедневного отчёта.
