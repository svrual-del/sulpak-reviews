# Sulpak Review Moderator

Автоматическая модерация отзывов на товары интернет-магазина [Sulpak](https://sulpak.kz) с помощью Claude AI.

## Как работает

```
Email (Exchange) → Парсинг HTML → Claude API → Модерация → Пометка/Публикация
```

1. Скрипт подключается к Exchange (EWS) и читает непрочитанные письма с отзывами
2. Парсит HTML-письмо, извлекая: имя, оценку, плюсы/минусы, текст, ссылки
3. Отправляет отзыв в Claude API для анализа
4. На основе ответа помечает письмо категорией в Exchange
5. При `AUTO_PUBLISH=true` — автоматически публикует одобренные отзывы

## Режимы работы

| Режим | `AUTO_PUBLISH` | Поведение |
|-------|---------------|-----------|
| **Помощник** | `false` | Только помечает письма категориями, ничего не публикует |
| **Автомат** | `true` | Публикует одобренные отзывы с confidence ≥ порога |

## Решения модерации

| Решение | Категория в Exchange | Описание |
|---------|---------------------|----------|
| `approve` | ✅ Одобрено ИИ | Отзыв о товаре, информативный, без нарушений |
| `reject` | ❌ Отклонено ИИ | Спам, нецензурщина, отзыв о сервисе, персональные данные |
| `manual_review` | ⚠️ Проверить вручную | Пограничный случай, жалоба на брак, неясный текст |

## Правила модерации

### Отклонить:
- Отзыв о сервисе/доставке, а не о товаре (>70% текста о сервисе)
- Спам, реклама, ссылки на сторонние сайты
- Нецензурная лексика, оскорбления
- Персональные данные (телефоны, адреса, ФИО сотрудников)
- Слишком короткий/неинформативный отзыв

### Одобрить:
- Описание опыта использования товара
- Конкретные характеристики, плюсы/минусы
- Допускается краткое упоминание сервиса (1-2 предложения)

### На ручную проверку:
- Пограничный контент (50/50 товар/сервис)
- Жалоба на брак/дефект
- Текст на другом языке или неразборчивый

## Установка

### Требования
- Python 3.11+
- Доступ к Exchange серверу (EWS)
- API ключ [Anthropic](https://console.anthropic.com)

### Шаги

```bash
# 1. Клонировать репозиторий
git clone https://github.com/<username>/sulpak-reviews.git
cd sulpak-reviews

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Создать .env файл
cp .env.example .env
# Заполнить данные в .env

# 4. Тестовый запуск
py sulpak_review_moderator.py
```

## Конфигурация (.env)

```env
# Exchange (EWS)
EWS_SERVER=mail2016.sulpak.kz
EWS_EMAIL=user@sulpak.kz
EWS_USERNAME=DOMAIN\username
EWS_PASSWORD=password
SKIP_SSL_VERIFY=true

# Anthropic API
ANTHROPIC_API_KEY=sk-ant-...

# Модерация
AUTO_PUBLISH=false
CONFIDENCE_THRESHOLD=0.85
REVIEW_FOLDER=Сайт отзывы на товар

# Отчёты
REPORT_TO=user@sulpak.kz
```

## Ежедневный отчёт

Скрипт `daily_report.py` собирает статистику из JSONL-логов и отправляет HTML-отчёт на email:

- Таблица: одобрено / отклонено / на проверку (с процентами)
- Причины отклонения (топ)
- Список отзывов на ручную проверку
- Список отклонённых с причинами

```bash
# Отчёт за сегодня
py daily_report.py

# Отчёт за конкретную дату
py daily_report.py 20260313
```

## Расписание (Windows Task Scheduler)

| Задание | Расписание | Описание |
|---------|-----------|----------|
| `SulpakReviewModeration` | Каждые 2 часа с 9:00 | Обработка новых отзывов |
| `SulpakDailyReport` | Ежедневно в 19:00 | Сводка за день на email |

### Управление

```cmd
:: Запустить модерацию вручную
schtasks /run /tn "SulpakReviewModeration"

:: Запустить отчёт вручную
schtasks /run /tn "SulpakDailyReport"

:: Посмотреть статус
schtasks /query /tn "SulpakReviewModeration"
schtasks /query /tn "SulpakDailyReport"
```

## Логи

- `moderation_YYYYMMDD.log` — полный лог работы (текстовый)
- `moderation_log_YYYYMMDD.jsonl` — результаты в JSON Lines (для отчётов)
